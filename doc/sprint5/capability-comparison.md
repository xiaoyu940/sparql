# RS Ontop Core vs 开源 Ontop 能力对比（Sprint5 基线）

> 更新时间: 2026-03-29  
> 对比版本: RS Ontop Core Sprint5 vs Ontop (官方开源版本)  
> 基于 Sprint4 完成后实际代码状态

---

## 1. 核心架构对比

| 能力项 | RS Ontop Core (Sprint5) | 开源 Ontop | 差距分析 |
|--------|------------------------|-----------|---------|
| **开发语言** | Rust + pgrx | Java | 语言差异，Rust 有内存安全优势 |
| **部署形式** | PostgreSQL 扩展 | 独立服务 + 可选嵌入 | RS Ontop 与 PG 深度集成 |
| **查询引擎** | 自定义 IR + SQL 生成 | Apache Calcite 优化器 | Ontop 优化器更成熟 |
| **SPARQL 支持** | 1.1 完整 + 扩展 | 1.1 完整 + 扩展 | **已基本对齐** |
| **R2RML 支持** | 部分实现 (映射定义) | 完整 R2RML + OWL | RS Ontop 映射功能较简单 |

## 8. 综合评估

### 8.1 与开源 Ontop 对比

| 维度 | Sprint4 前 | Sprint4 后 | 目标 |
|------|-----------|-------------|------|
| **SPARQL 完整度** | 75% | **90%** | 95% |
| **查询优化** | 80% | **85%** | 90% |
| **推理能力** | 10% | 10% | 50% |
| **数据源支持** | 20% | 20% | 40% |
| **生产就绪度** | 65% | **80%** | 90% |

### 8.2 核心优势

1. **PostgreSQL 原生集成** - 作为扩展部署，无需额外服务
2. **内存安全** - Rust 语言保证
3. **流式查询** - 支持大结果集
4. **查询缓存** - Sprint4 新增，性能优化
5. **属性路径** - Sprint4 新增，支持复杂导航模式
6. **完整分页支持** - ORDER BY + LIMIT + OFFSET 组合

### 8.3 核心差距

1. **推理能力** - 无 OWL/RDFS 支持（最大差距）
2. **多数据源** - 仅 PostgreSQL
3. **联邦查询执行** - SERVICE 框架就绪，HTTP 执行待完善
4. **命名图** - GRAPH 模式未实现

## 10. 结论

**Sprint4 状态**: 
- **ORDER BY / OFFSET**: 100% 完成，SQL 生成已对齐
- **属性路径**: 100% 完成，支持全部 7 种路径类型
- **查询缓存**: 100% 完成，含统计与 PG 外部函数
- **SERVICE**: 50% 完成，框架就绪待 HTTP 实现

**与开源 Ontop 差距**: 
- **基础查询能力**: 从 75% → **90%** (+15%)
- **生产就绪度**: 从 65% → **80%** (+15%)
- **核心差距项**: OWL 推理、命名图、多数据源

**建议 Sprint5+ 重点**:
1. OWL 2 QL 推理（高价值，提升语义能力）
2. 命名图 GRAPH 支持（SPARQL 1.1 合规）
3. SERVICE 完整 HTTP 实现（联邦查询执行）
4. SPARQL 函数库扩展（兼容性提升）
