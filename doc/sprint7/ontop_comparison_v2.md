# RS Ontop Core vs Ontop 开源系统功能对比 (已更新)

> 更新时间：2026-04-01  
> 对比版本：Ontop 5.5.0 (2026-02-14)  
> 本次更新：CONSTRUCT/ASK/DESCRIBE 已实现

---

## 1. 总体对比概览

| 维度 | RS Ontop Core (当前) | Ontop 开源 | 差距 |
|------|---------------------|-----------|------|
| **架构** | PostgreSQL 扩展 (Rust) | Java 独立应用 | 不同架构 |
| **数据库支持** | PostgreSQL | 15+ 种数据库 | ⚠️ 差距大 |
| **SPARQL 协议** | SELECT/CONSTRUCT/ASK/DESCRIBE | SPARQL 1.1 完整 | ✅ 接近 |
| **映射语言** | R2RML 子集 | R2RML + Ontop ML | ⚠️ 部分差距 |
| **推理能力** | OWL 2 QL 饱和 | OWL 2 QL 完整 | ✅ 接近 |
| **优化器** | DPSize + 基础 | 企业级优化器 | ⚠️ 中等差距 |
| **部署方式** | PG 扩展 | CLI / Endpoint / Library | ⚠️ 差距大 |

## 3. 核心差距分析

### 3.1 🔴 严重缺失（必须实现）

1. **SPARQL 子查询支持**
   - 嵌套 SELECT 查询
   - 子查询中的聚合

2. **多数据库方言**
   - 至少支持 MySQL、SQLite
   - SQL 方言抽象层

3. **部署方式**
   - CLI 工具
   - 独立 HTTP 服务
   - Docker 官方镜像

### 3.2 🟡 中等差距（建议实现）

1. **R2RML 完整规范**
   - rr:Join 支持
   - 复杂 SQL 查询作为逻辑表

2. **高级优化**
   - 查询并行化
   - 更精确的基数估计

3. **工具链**
   - Protégé 插件
   - 映射编辑工具

### 3.3 🟢 接近平齐

1. **SPARQL 查询类型** - CONSTRUCT/ASK/DESCRIBE ✅ **已实现**
2. **OWL 2 QL 推理** - 饱和器完整
3. **JOIN 优化** - DPSize 已集成
4. **R2RML 基础** - 解析和转换已完成
5. **PostgreSQL 支持** - 作为扩展原生支持

## 4. 技术架构差异

### 4.1 架构对比

| 维度 | RS Ontop Core | Ontop |
|------|--------------|-------|
| **语言** | Rust | Java |
| **运行时** | PostgreSQL 扩展 | JVM |
| **部署** | 嵌入式 (PG) | 独立/嵌入式 |
| **性能** | 原生性能 | JVM 性能 |
| **生态** | 新兴 | 成熟 |
| **维护** | 单一作者 | 社区+企业 |

### 4.2 各自优势

**RS Ontop Core 优势：**
- 作为 PG 扩展，无缝集成数据库
- Rust 内存安全和性能
- 无 JVM 依赖
- CONSTRUCT/ASK/DESCRIBE 实现简洁

**Ontop 开源优势：**
- 生态成熟，企业验证
- 多数据库支持
- 完整工具链
- 子查询和高级 SPARQL 特性
